## Background: The Interior Stencil Equation

From the math reference (Section 1), the general implicit finite difference formula for the nu-th derivative on a uniform grid with spacing h is:

```
sum_{k=-s}^{s} delta_k * f^(nu)_{i+k} = (1/h^nu) * sum_{j=-p}^{p} gamma_j * f_{i+j}
```

Conventions:
- `delta_0 = 1` (normalization)
- `delta_{-k} = delta_k` (LHS symmetry for all derivatives)
- nu=1 (1st derivative): `gamma_{-j} = -gamma_j`, `gamma_0 = 0` (antisymmetry)
- nu=2 (2nd derivative): `gamma_{-j} = gamma_j` (symmetry), `gamma_0` is derived

The independent unknowns are `gamma_1, ..., gamma_p` and `delta_1, ..., delta_s`. The system is solved by Taylor-expanding both sides about point i and matching coefficients of `f^(m)_i` through the required order.

Scheme families:
- **E (explicit):** s=0, no LHS coupling. Order = 2p.
- **T (tridiagonal compact):** s=1, tridiagonal LHS. Order = 2(p+s).

**`StencilSpec` frozen dataclass:**
```python
from dataclasses import dataclass
from sympy import Rational

@dataclass(frozen=True)
class StencilSpec:
    s: int       # LHS half-bandwidth (0 = explicit, 1 = tridiagonal)
    p: int       # RHS half-bandwidth (number of points on each side)
    nu: int      # derivative order (1 or 2)
```

Properties to implement:
- `order -> int`: Returns `2 * (self.p + self.s)` -- the formal accuracy order of the interior scheme.
- `n_unknowns -> int`: Returns `self.p + self.s` -- the number of independent unknowns (gamma_1..gamma_p and delta_1..delta_s).
- `scheme_name -> str`: Returns a human-readable name string. Rules:
  - If `s == 0`: `f"E{self.order}_{self.nu}"` (e.g., `"E4_1"`, `"E2_2"`)
  - If `s == 1`: `f"T{self.order}_{self.nu}"` (e.g., `"T4_1"`, `"T6_1"`)
  - If `s >= 2`: `f"P{self.order}_{self.nu}"` (pentadiagonal, not currently used)

**`InteriorCoefficients` frozen dataclass:**
```python
@dataclass(frozen=True)
class InteriorCoefficients:
    spec: StencilSpec
    gamma: dict[int, Rational]  # {1: Rational(2,3), 2: Rational(-1,12), ...}
    delta: dict[int, Rational]  # {1: Rational(1,4), ...} -- empty for explicit
```

The `gamma` dict maps positive index j (1..p) to the coefficient value. Negative indices are inferred from symmetry. For nu=2, `gamma_0` is NOT stored in this dict -- it is derived.

The `delta` dict maps positive index k (1..s) to the coefficient value. Empty dict `{}` for explicit schemes (s=0).

**Helper function `full_gamma_array`:**
```python
def full_gamma_array(coeffs: InteriorCoefficients) -> list[Rational]:
```
Returns a list of length `2*p + 1` representing `[gamma_{-p}, ..., gamma_0, ..., gamma_p]`.
- For nu=1: `gamma_0 = 0`, `gamma_{-j} = -gamma_j`
- For nu=2: `gamma_0 = -2 * sum(gamma_j for j in 1..p)` (from the constraint that `sum of all gamma_j = 0`), `gamma_{-j} = gamma_j`
- Index mapping: position `i` in the returned list corresponds to `j = i - p` (so `gamma_{-p}` is at index 0, `gamma_0` is at index p, `gamma_p` is at index 2p).

**Helper function `full_delta_array`:**
```python
def full_delta_array(coeffs: InteriorCoefficients) -> list[Rational]:
```
Returns a list of length `2*s + 1` representing `[delta_{-s}, ..., delta_0, ..., delta_s]`.
- `delta_0 = 1` always.
- `delta_{-k} = delta_k` always.
- For s=0: returns `[Rational(1)]`.
- Index mapping: position `i` corresponds to `k = i - s`.

**Verification criteria:**

### 20.2b -- Taylor series solver (`interior.py`)

Add the `derive_interior` function to the same file (~120 lines, bringing total to ~200 lines).

```python
def derive_interior(s: int, p: int, nu: int) -> InteriorCoefficients:
```

**Algorithm, step by step:**

**Step 1 -- Create symbolic unknowns.**
```python
from sympy import symbols, Rational, Matrix, linsolve, factorial

gamma_syms = symbols(f'gamma_1:{p+1}')   # gamma_1, gamma_2, ..., gamma_p
delta_syms = symbols(f'delta_1:{s+1}')   # delta_1, ..., delta_s (empty tuple if s=0)
unknowns = list(gamma_syms) + list(delta_syms)
```

**Step 2 -- Set up the Taylor matching equations.**

Taylor-expand both sides of the interior stencil equation about point i. For each Taylor order `m` from 0 to `2*(p+s) - 1`, the coefficient of `f^(m)_i` on each side must match:

- **RHS coefficient of `f^(m)_i`:** This comes from expanding `gamma_j * f_{i+j}`:
  ```
  rhs_coeff(m) = sum_{j=-p}^{p} gamma_j * j^m / m!
  ```
  Using symmetry to express in terms of unknowns only:
  - nu=1 (antisymmetric gamma): `gamma_{-j} = -gamma_j`, so `sum = 2 * sum_{j=1}^{p} gamma_j * j^m / m!` for odd m, and `sum = 0` for even m (terms cancel).
  - nu=2 (symmetric gamma): `gamma_{-j} = gamma_j`, so `sum = gamma_0 / m! * 0^m + 2 * sum_{j=1}^{p} gamma_j * j^m / m!` for even m, and `sum = 0` for odd m. The `0^m` term contributes only for m=0.

- **LHS coefficient of `f^(m)_i`:** This comes from expanding `delta_k * f^(nu)_{i+k}`, which contributes to `f^(m)_i` only when `m >= nu`:
  ```
  lhs_coeff(m) = sum_{k=-s}^{s} delta_k * k^(m-nu) / (m-nu)!    [for m >= nu]
  lhs_coeff(m) = 0                                                  [for m < nu]
  ```
  Using symmetry (`delta_{-k} = delta_k`):
  - For even `(m - nu)`: `lhs = 1/(m-nu)! * (0^(m-nu) + 2 * sum_{k=1}^{s} delta_k * k^(m-nu))`
  - For odd `(m - nu)`: `lhs = 0` (terms cancel by symmetry)
  - The `0^(m-nu)` term equals 1 when `m = nu` (contributing `delta_0 = 1`), and 0 otherwise.

**Step 3 -- Assemble equations.**

For each `m = 0, 1, 2, ..., 2(p+s) - 1`:
- Compute `lhs_coeff(m)` and `rhs_coeff(m)` as symbolic expressions in the unknowns.
- The equation is: `rhs_coeff(m) - lhs_coeff(m) = 0`.

**Step 4 -- Filter trivial equations.**

Many equations reduce to `0 = 0` due to parity cancellation. Only non-trivial equations survive:

- **nu=1 (1st derivative):**
  - Odd m: RHS is nonzero (gamma terms survive antisymmetry). LHS is 0 when `m - 1` is odd (i.e., m is even), nonzero when `m - 1` is even (i.e., m is odd). So for odd m: both sides contribute -- these are the "mixed" equations involving both gamma and delta unknowns.
  - Even m > 0: RHS is 0 (gamma cancels). LHS is nonzero when `m - 1` is odd -- which requires m even, so `m - 1` is odd, yes. These give pure delta equations.
  - Even m = 0: RHS = 0 (m=0, even, gamma cancels). LHS = 0 (m < nu=1). Trivial.
  - **Summary for nu=1:** Non-trivial equations come from odd m (giving gamma+delta equations) and even m >= 2 where m-1 is odd (giving pure delta equations). In practice, for s=0 (explicit), only the odd-m equations survive, giving exactly p equations in p unknowns. For s=1 (tridiagonal), even m = 2, 4, ... also contribute delta equations.

- **nu=2 (2nd derivative):**
  - Even m: RHS is nonzero (gamma terms survive symmetry). LHS is nonzero when `m - 2` is even. For even m, `m - 2` is even, so both sides contribute. These equations involve both gamma and delta.
  - Odd m: RHS is 0 (gamma cancels by symmetry). LHS is nonzero when `m - 2` is odd, which requires odd m. These give pure delta equations.
  - m = 0: RHS = `gamma_0 + 2*sum(gamma_j)`. LHS = 0 (m < nu=2). This gives the constraint `gamma_0 = -2*sum(gamma_j)` -- but gamma_0 is NOT an unknown. For the solver, this equation is automatically satisfied if we define gamma_0 from the constraint, so we skip m=0.
  - m = 1: RHS = 0 (odd, cancels). LHS = 0 (m < nu=2). Trivial.
  - **Summary for nu=2:** Non-trivial equations come from even m >= 2 (gamma+delta equations) and odd m >= 3 (pure delta equations). For s=0 (explicit), only even m >= 2 survives, giving p equations (m = 2, 4, ..., 2p) -- but the m=2 equation is `1 = 1` (from the leading derivative), so effectively p-1 equations from m = 4, 6, ..., 2p plus the m=2 identity. Actually, m=2 gives: `LHS = delta_0 = 1`, `RHS = 2*sum(gamma_j * j^2 / 2) = sum(gamma_j * j^2)`. So the equation is `sum(gamma_j * j^2) = 1`, which is a non-trivial constraint on the gamma unknowns.

**Step 5 -- Build the linear system matrix and solve.**

Collect the non-trivial equations into an augmented matrix `[A | b]` where `A * unknowns = b`. The system has exactly `p + s` equations and `p + s` unknowns (the system is square and has a unique solution).

```python
from sympy import zeros

n = p + s  # number of unknowns = number of non-trivial equations
A = zeros(n, n)
b = zeros(n, 1)
# ... fill from the equation coefficients ...
solution = linsolve((A, b), unknowns)
```

Extract the unique solution as a dict mapping each symbol to its `Rational` value.

**Step 6 -- Construct and return `InteriorCoefficients`.**

```python
gamma_dict = {j: sol[gamma_syms[j-1]] for j in range(1, p+1)}
delta_dict = {k: sol[delta_syms[k-1]] for k in range(1, s+1)}
return InteriorCoefficients(
    spec=StencilSpec(s, p, nu),
    gamma=gamma_dict,
    delta=delta_dict,
)
```

**Implementation notes:**

- Use `sympy.Rational` for all arithmetic -- never introduce floats.
- Use `sympy.linsolve()` (not `solve()`) for the linear system -- it is 12x faster per the performance rules in the parent plan.
- Handle the edge case where `s = 0`: `delta_syms` is an empty tuple, the delta dict is empty, and all LHS terms reduce to just `delta_0 = 1`.
- For nu=2, `gamma_0` is NOT an unknown. It is derived after solving via `gamma_0 = -2 * sum(gamma[j] for j in 1..p)`. This is computed inside `full_gamma_array`, not during the solve.
- The function should validate inputs: `s >= 0`, `p >= 1`, `nu in {1, 2}`. Raise `ValueError` for invalid inputs.

#### Test group 4: Polynomial exactness

For a stencil of order `q = 2(p + s)` approximating the nu-th derivative, the stencil must be exact for all polynomials of degree `d = 0, 1, ..., q + nu - 1`. That is, when applied to `f(x) = x^d`, it must produce exactly `f^(nu)(x) = d!/(d-nu)! * x^(d-nu)` (or 0 for `d < nu`).

Test method: pick `x_i = 0` (center point), grid spacing `h = 1`. For each degree d, evaluate:
```
LHS = sum_{k=-s}^{s} delta_k * exact_derivative(d, nu, k*h)
RHS = (1/h^nu) * sum_{j=-p}^{p} gamma_j * f(j*h)
```
where `f(x) = x^d` and `exact_derivative(d, nu, x) = d!/(d-nu)! * x^(d-nu)` if `d >= nu`, else 0.

Assert `LHS == RHS` exactly (rational arithmetic).

```python
@pytest.mark.parametrize("s, p, nu", [
    (0, 1, 1), (0, 2, 1), (0, 3, 1), (0, 4, 1),  # E2, E4, E6, E8
    (1, 1, 1), (1, 2, 1), (1, 3, 1),               # T4, T6, T8
    (0, 1, 2), (0, 2, 2),                           # E2_2, E4_2
])
def test_polynomial_exactness(s, p, nu):
    coeffs = derive_interior(s, p, nu)
    gamma = full_gamma_array(coeffs)
    delta = full_delta_array(coeffs)
    order = 2 * (p + s)
    max_degree = order + nu - 1

    for d in range(0, max_degree + 1):
        # RHS: (1/h^nu) * sum gamma_j * j^d  (h=1)
        rhs = sum(
            gamma[j_idx] * Rational(j**d)
            for j_idx, j in enumerate(range(-p, p + 1))
        )
        # LHS: sum delta_k * d!/(d-nu)! * k^(d-nu)  if d >= nu, else 0
        if d >= nu:
            exact_factor = Rational(factorial(d), factorial(d - nu))
            lhs = sum(
                delta[k_idx] * exact_factor * Rational(k**(d - nu))
                for k_idx, k in enumerate(range(-s, s + 1))
            )
        else:
            lhs = Rational(0)
        assert rhs == lhs, f"Failed for degree {d}: RHS={rhs}, LHS={lhs}"
```

Also verify that the stencil is NOT exact for degree `max_degree + 1` (the leading truncation error term is nonzero). This confirms the order is exactly `2(p+s)` and not accidentally higher.

```python
    # Verify NOT exact at degree max_degree + 1 (confirms order is tight)
    d = max_degree + 1
    rhs = sum(
        gamma[j_idx] * Rational(j**d)
        for j_idx, j in enumerate(range(-p, p + 1))
    )
    if d >= nu:
        exact_factor = Rational(factorial(d), factorial(d - nu))
        lhs = sum(
            delta[k_idx] * exact_factor * Rational(k**(d - nu))
            for k_idx, k in enumerate(range(-s, s + 1))
        )
    else:
        lhs = Rational(0)
    assert rhs != lhs, f"Stencil is unexpectedly exact at degree {d}"
```
