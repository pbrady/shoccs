coefficients for explicit schemes (E4u, E6u, E8u first derivative). Given scheme
parameters (p, s, nu), produce boundary coefficients as closed-form rational
expressions of free alpha parameters, then solve conservation (SBP) constraints

## Scheme Parameters Reference

These constants are derived from the formulas in the math reference (Section 2)
and must match the C++ `info` structs exactly.

| Scheme | s | p | r | t=r+p | q=2(p+s)-1 | n_alpha | Interior order |
|--------|---|---|---|-------|------------|---------|----------------|
| E4u_1  | 0 | 2 | 3 | 5     | 3          | 2       | 4              |
| E6u_1  | 0 | 3 | 5 | 8     | 5          | 5       | 6              |
| E8u_1  | 0 | 4 | 7 | 11    | 7          | 7       | 8              |

Where:
- `s` = LHS bandwidth (0 for explicit schemes)
- `p` = RHS half-bandwidth (interior stencil uses points -p..p)
- `r` = number of boundary rows
- `t` = boundary stencil width (number of grid points each boundary row touches)
- `q` = polynomial order of the boundary scheme (one less than interior)
- `n_alpha` = total free parameters after Taylor matching + conservation

    ic = S.Zero
    for m in range(max(0, j - r - p), j - r + p + 1):

### 20.3a -- Taylor system builder ✅ DONE

**File:** `scripts/stencil_gen/stencil_gen/taylor_system.py`
**Test:** `cd scripts/stencil_gen && uv run pytest tests/test_boundary.py -v -k "taylor"`

Implement `build_taylor_system(i, t, q, nu=1)` that constructs the
Vandermonde-like linear system from Section 2 of the math reference.

**Function signature:**

```python
def build_taylor_system(
    i: int,       # row index (0..r-1), the grid point being approximated
    t: int,       # boundary stencil width (number of columns)
    q: int,       # polynomial order of boundary scheme
    nu: int = 1,  # derivative order (1 for first derivative)
) -> tuple[Matrix, Matrix]:
    """Return (V, rhs) where V is (q+1) x t and rhs is (q+1) x 1."""
```

**Vandermonde-like construction:**

For an explicit scheme (s=0), the boundary approximation at row i is:

    f'_i ~ (1/h) * sum_{j=0}^{t-1} gamma_{i,j} * f_j

Taylor-expanding f_j about x_i (on a unit grid, h=1):

    f_j = sum_{k=0}^{inf} (j - i)^k / k! * f^(k)_i

Matching coefficients of f^(k)_i for k = 0, 1, ..., q yields the system V * gamma = rhs:

    V[k, j] = (j - i)^k / k!     for k = 0..q, j = 0..t-1
    rhs[k]  = 1  if k == nu
    rhs[k]  = 0  otherwise

The matrix V has shape (q+1) x t. Since t > q+1 for all schemes in our
inventory, the system is underdetermined and has n_free = t - (q+1) free
parameters per row.

**Worked example -- E4u_1 row 0 (i=0, t=5, q=3, nu=1):**

V is 4x5:

```
k=0: [  1      1      1      1      1    ]    (j-0)^0 / 0!
k=1: [  0      1      2      3      4    ]    (j-0)^1 / 1!
k=2: [  0    1/2      2    9/2      8    ]    (j-0)^2 / 2!
k=3: [  0    1/6      4/3  9/2    32/3   ]    (j-0)^3 / 3!
```

rhs = [0, 1, 0, 0]^T (because nu=1, so only the k=1 entry is 1).

This gives 4 equations in 5 unknowns => 1 free parameter for row 0.

**Implementation notes:**
- Use `sympy.Rational` for all arithmetic (never Python floats).
- Use `sympy.Matrix` for V and rhs.
- The factorial in the denominator is `sympy.factorial(k)`.
- Return plain Matrix objects; the caller decides how to partition/solve.

**Test:**
- Verify V shape is (4, 5) for E4u_1 row 0.
- Verify V entries match the worked example above.
- Verify rhs = Matrix([0, 1, 0, 0]).
- Spot-check V for E4u_1 row 1 (i=1): V[1,0] = -1, V[1,2] = 1, etc.
- Spot-check V for E8u_1 row 0 (i=0, t=11, q=7): shape is (8, 11).

### 20.3b -- Single-row boundary solver ✅ DONE

**File:** `scripts/stencil_gen/stencil_gen/boundary.py`
**Test:** `cd scripts/stencil_gen && uv run pytest tests/test_boundary.py -v -k "solve_row"`
**Depends on:** 20.3a (`solve_boundary_row` calls `build_taylor_system`).

Implement `solve_boundary_row()` that takes the Taylor system for a single
boundary row and solves it, expressing the t coefficients as symbolic functions
of free alpha parameters.

**Function signature:**

```python
@dataclass
class BoundaryRow:
    row_index: int                      # i (0..r-1)
    coefficients: list[Expr]            # length t, each is Expr in alpha symbols
    free_params: list[Symbol]           # the alpha symbols for this row

def solve_boundary_row(
    i: int,           # row index
    t: int,           # stencil width
    q: int,           # polynomial order
    nu: int,          # derivative order
    free_symbols: list[Symbol],  # pre-created alpha symbols for this row
) -> BoundaryRow:
```

**Algorithm:**

1. Call `build_taylor_system(i, t, q, nu)` to get `(V, rhs)`.

2. The system has n_free = t - (q+1) free parameters. The caller provides
   exactly n_free symbols in `free_symbols`.

3. **Column partitioning convention:** The LAST n_free columns of V are the
   free columns. This matches the existing C++ code where the last coefficient(s)
   in each row are the raw alpha values:
   - E4u_1 row 0: c[4] = alpha[0] (column index 4 is the last of 5 columns)
   - E4u_1 row 1: c[9] = alpha[1] (column index 4 of row 1)
   - E6u_1 row 3: c[30] = alpha[3], c[31] = alpha[4] (last 2 of 8 columns)
   - E8u_1 row 5: c[63] = alpha[5], c[64] = alpha[6] (last 2 of 11 columns)

4. Partition V = [V_det | V_free] where:
   - V_det is (q+1) x (q+1) -- the first q+1 columns (square, invertible)
   - V_free is (q+1) x n_free -- the last n_free columns

5. Set the free coefficients equal to their alpha symbols:
   alpha_vec = Matrix(free_symbols)

6. Solve for the determined coefficients:
   V_det * gamma_det = rhs - V_free * alpha_vec
   Use `sympy.linsolve()` (or equivalently `V_det.solve(rhs - V_free * alpha_vec)`)
   to get gamma_det as expressions of the alpha symbols.

7. Concatenate: coefficients = list(gamma_det) + list(free_symbols).

8. Call `cancel()` on each coefficient expression to normalize rational
   functions. Do NOT call `simplify()` (too slow for larger schemes).

**Free parameter naming convention:**

The alpha symbols are shared across rows and named `alpha_0, alpha_1, ...`
with a global index matching the C++ `alpha[]` array:

| Scheme | Row | n_free | Alpha indices | C++ mapping |
|--------|-----|--------|---------------|-------------|
| E4u_1  | 0   | 1      | alpha_0       | c[4] = alpha[0] |
| E4u_1  | 1   | 1      | alpha_1       | c[9] = alpha[1] |
| E4u_1  | 2   | 1      | (constrained) | c[14] = f(alpha_0, alpha_1) |
| E6u_1  | 0   | 2      | alpha_0       | c[6] = alpha[0], c[7] = 0 |
| E6u_1  | 1   | 2      | alpha_1       | c[14] = alpha[1], c[15] = 0 |
| E6u_1  | 2   | 2      | alpha_2       | c[22] = alpha[2], c[23] = 0 |
| E6u_1  | 3   | 2      | alpha_3, alpha_4 | c[30] = alpha[3], c[31] = alpha[4] |
| E6u_1  | 4   | 2      | (constrained) | c[38..39] = f(alpha_0..4) |
| E8u_1  | 0   | 3      | alpha_0       | c[8] = alpha[0], c[9..10] = 0 |
| E8u_1  | 1   | 3      | alpha_1       | c[19] = alpha[1], c[20..21] = 0 |
| E8u_1  | 2   | 3      | alpha_2       | c[30] = alpha[2], c[31..32] = 0 |
| E8u_1  | 3   | 3      | alpha_3       | c[41] = alpha[3], c[42..43] = 0 |
| E8u_1  | 4   | 3      | alpha_4       | c[52] = alpha[4], c[53..54] = 0 |
| E8u_1  | 5   | 3      | alpha_5, alpha_6 | c[63] = alpha[5], c[64] = alpha[6], c[65] = 0 |
| E8u_1  | 6   | 3      | (constrained) | c[74..76] = f(alpha_0..6) |

**Observation about zero-valued free slots:** For rows 0..(r-2) of E6u and
E8u, each row has n_free free columns, but only ONE alpha is truly free -- the
remaining (n_free - 1) free parameters are implicitly set to zero. This is
handled by the caller: pass `[alpha_k, S.Zero, S.Zero, ...]` as `free_symbols`
for rows that only use one free alpha, and `[alpha_k, alpha_{k+1}]` plus zeros
for the penultimate row. The last row (r-1) is constrained by conservation
(20.3d) rather than being solved here.

**Worked example -- E4u_1 row 0 solution:**

V_det (4x4, columns 0..3):
```
[  1      1      1      1   ]
[  0      1      2      3   ]
[  0    1/2      2    9/2   ]
[  0    1/6    4/3    9/2   ]
```

V_free (4x1, column 4):
```
[    1   ]
[    4   ]
[    8   ]
[  32/3  ]
```

Solving V_det * gamma_det = [0, 1, 0, 0]^T - V_free * [alpha_0]:

```
gamma_det[0] = (6*alpha_0 - 11) / 6
gamma_det[1] = 3 - 4*alpha_0
gamma_det[2] = (12*alpha_0 - 3) / 2
gamma_det[3] = -(12*alpha_0 - 1) / 3
```

So the full row 0 is: [(6*alpha_0 - 11)/6,  3 - 4*alpha_0,  (12*alpha_0 - 3)/2,  -(12*alpha_0 - 1)/3,  alpha_0].

Cross-check against E4u_1.cpp lines 80-84: exact match.

### 20.3c -- Full boundary derivation driver ✅ DONE

**File:** `scripts/stencil_gen/stencil_gen/boundary.py` (extend)
**Test:** No dedicated test cases. `derive_boundary()` is an orchestrator whose

(E6u), and 20.3g (E8u). Specifically, 20.3e test cases 2–4 verify that each
row's symbolic coefficients match the C++ code, which proves both

**Blocking dependency:** Phase 20.2 must be completed first -- this item
imports `from stencil_gen.interior import derive_interior, full_gamma_array`.
If 20.2 is not yet done, 20.3a and 20.3b can still proceed independently.

Implement `derive_boundary()` that orchestrates the full boundary derivation
for a given scheme.

**Function signatures:**

```python
@dataclass
class BoundaryResult:
    r: int                      # number of boundary rows
    t: int                      # boundary stencil width
    rows: list[BoundaryRow]     # r BoundaryRow objects
    interior_coeffs: list[Rational]  # 2p+1 interior coefficients (from 20.2)
    all_free_params: list[Symbol]    # all alpha symbols, globally ordered

def derive_boundary(
    p: int,          # RHS half-bandwidth
    nu: int = 1,     # derivative order
    s: int = 0,      # LHS bandwidth (0 for explicit)
) -> BoundaryResult:
```

**Algorithm:**

1. Compute derived parameters:
   - `r = 2*p - 1` (number of boundary rows for explicit first-derivative)
   - `t = r + p` (boundary stencil width)
   - `q = 2*(p + s) - 1` (boundary polynomial order)
   - `n_free_per_row = t - (q + 1)` (free parameters in each row's Taylor system)

2. Determine total number of free alphas and their row assignment. For
   explicit 1st-derivative schemes (s=0, nu=1):

   - `n_free_per_row = t - (q + 1) = p - 1`
   - Rows 0 through r-3: each row uses 1 active alpha, rest zero-padded
   - Row r-2 (penultimate): uses `min(n_free_per_row, 2)` active alphas,
     rest zero-padded. When n_free_per_row = 1 (E4u), this is just 1.
     When n_free_per_row >= 2 (E6u, E8u), this is 2.
   - Row r-1 (last): fully constrained by conservation (no new alphas)
   - `n_alpha = (r - 2) * 1 + min(n_free_per_row, 2)`

   Verification:
   - E4u (p=2): n_free_per_row=1, n_alpha = 1*1 + 1 = 2 ✓
   - E6u (p=3): n_free_per_row=2, n_alpha = 3*1 + 2 = 5 ✓
   - E8u (p=4): n_free_per_row=3, n_alpha = 5*1 + 2 = 7 ✓

3. Create global alpha symbols: `alpha_0, alpha_1, ..., alpha_{n_alpha-1}`.

4. For each row i = 0, ..., r-2, track `alpha_idx` starting at 0:
   - If i < r-2 (rows 0 through r-3):
     `free_symbols = [alpha_{alpha_idx}, S.Zero, ..., S.Zero]` (length n_free_per_row)
     Increment `alpha_idx` by 1.
   - If i == r-2 (penultimate row):
     `n_active = min(n_free_per_row, 2)`
     `free_symbols = [alpha_{alpha_idx}, ..., alpha_{alpha_idx+n_active-1}, S.Zero, ...]`
     (length n_free_per_row, zero-padded after n_active entries)
     Increment `alpha_idx` by n_active.
   - Call `solve_boundary_row(i, t, q, nu, free_symbols)`.

5. For row r-1: create n_free_per_row placeholder symbols named
   `phi_0, phi_1, ..., phi_{n_free_per_row - 1}`. Unlike earlier rows, ALL
   n_free_per_row slots are active (no zero-padding): pass
   `[phi_0, phi_1, ..., phi_{n_free_per_row - 1}]` as `free_symbols`.
   Call `solve_boundary_row(i, t, q, nu, free_symbols)`. These placeholder
   symbols are NOT added to `all_free_params`; they will be solved for by
   conservation in 20.3d.

   Placeholder counts per scheme:
   - E4u: 1 placeholder (phi_0)
   - E6u: 2 placeholders (phi_0, phi_1)
   - E8u: 3 placeholders (phi_0, phi_1, phi_2)

6. Retrieve interior coefficients from `derive_interior(s, p, nu)` (20.2).
   **Dependency:** 20.2 must be completed first; the import path is
   `from stencil_gen.interior import derive_interior`.

7. Return `BoundaryResult` with all collected data.

**Implementation notes:**
- The zero-padding of free_symbols (setting trailing free params to zero) is
  what produces the `c[T-1] = 0` entries visible in E6u and E8u. In E4u, each
  row has only 1 free parameter, so no zero-padding is needed.
- The driver does NOT solve conservation -- that is step 20.3d.
- The last row's coefficients will be expressions of temporary placeholder
  symbols that 20.3d will solve for.

### 20.3d -- Conservation constraint solver

**File:** `scripts/stencil_gen/stencil_gen/conservation.py`
**Test:** `cd scripts/stencil_gen && uv run pytest tests/test_boundary.py -v -k "conservation"`

Implement the conservation (SBP) constraint system from Section 3 of the math
reference.

**Function signatures:**

```python
def build_conservation_system(
    r: int,
    t: int,
    p: int,
    boundary_rows: list[BoundaryRow],  # r rows from 20.3b/c
    interior_coeffs: list[Rational],   # 2p+1 interior coefficients
) -> tuple[list[Expr], list[Symbol], list[Symbol]]:
    """Return (equations, w_symbols, last_row_free_symbols).

    equations: list of expressions that must equal zero
    w_symbols: [w_0, w_1, ..., w_{r-1}] quadrature weight symbols
    last_row_free_symbols: placeholder symbols from last boundary row
    """

def solve_conservation(
    equations: list[Expr],
    w_symbols: list[Symbol],
    last_row_free: list[Symbol],
    all_free_params: list[Symbol],
    boundary_rows: list[BoundaryRow],  # r rows from derive_boundary
) -> tuple[dict[Symbol, Expr], list[BoundaryRow]]:
    """Return (solution_dict, updated_rows).

    solution_dict maps each w_i and each phi_j to its value as an
    expression of the unconstrained alpha symbols:
      - w_0 -> Expr(alpha_0, ..., alpha_{n-1})
      - ...
      - w_{r-1} -> Expr(alpha_0, ..., alpha_{n-1})
      - phi_0 -> Expr(alpha_0, ..., alpha_{n-1})
      - ...
      - phi_{n_free-1} -> Expr(alpha_0, ..., alpha_{n-1})

    updated_rows is a list[BoundaryRow] of length r: rows 0..r-2 are
    copied unchanged from the input boundary_rows; row r-1 has its
    placeholder symbols substituted with the solved expressions.
    """
```

**Algorithm for `build_conservation_system`:**

The conservation constraint (SBP property, Eq. 15, 2019) requires that the
composite operator Q = HD (where H = diag(w_0, ..., w_{r-1}, 1, 1, ...) is
the diagonal norm matrix) satisfies Q + Q^T = diag(-1, 0, ..., 0, 1). Since
every row of D sums to zero (exact for constants), each row of Q also sums to
zero, so the column sums of Q equal the diagonal of the boundary matrix:
column 0 sums to -1, all other columns sum to 0.

The math reference (Section 3, line "r + p - 1 additional linear constraints")
gives the total conservation constraint count as r + p - 1 = t - 1. Note that
the column range "i = r, ..., t-1" cited in the math reference describes only
the overlap columns; the full system also includes columns 0 through r-1.

For the left boundary block:

1. Construct the conceptual matrix B of shape r x t, where B[i, j] is the
   j-th coefficient of boundary row i (these are symbolic expressions from
   20.3b/c).

2. Compute the interior stencil contribution IC(j) to each column j.
   `interior_coeffs` is the 2p+1-length array from
   `full_gamma_array(derive_interior(s, p, nu))` (Phase 20.2), indexed as
   `interior_coeffs[k]` = gamma_{k-p} (so index 0 = gamma_{-p}, index p =
   gamma_0, index 2p = gamma_p). Interior row r+m (m >= 0) is centered at
   grid point r+m and covers columns (r+m-p)..(r+m+p). Its coefficient at
   column j is `interior_coeffs[j - (r+m) + p]`.

   IC(j) = sum of `interior_coeffs[j - (r+m) + p]` over all valid m, where
   valid m satisfies: m >= 0 AND |j - (r+m)| <= p, i.e.,
   `max(0, j - r - p) <= m <= j - r + p`.

   If `j < r - p` (equivalently `j < p - 1`, since `r = 2p - 1`): no interior
   rows reach column j, so IC(j) = 0.

   Concrete first-column-with-IC for each scheme:
   - E4u (p=2): IC(0) = 0; IC(1) = gamma[-2] = 1/12
   - E6u (p=3): IC(0) = IC(1) = 0; IC(2) = gamma[-3] = -1/60
   - E8u (p=4): IC(0) = IC(1) = IC(2) = 0; IC(3) = gamma[-4] = 1/280

3. Build one equation per column j = 0, 1, ..., t-2. Each equation is a
   SymPy Expr that must equal zero:

   **Column 0** (IC(0) = 0 for all schemes with p >= 2):

       w_0 * B[0,0] + w_1 * B[1,0] + ... + w_{r-1} * B[r-1,0] + 1 = 0

   The "+1" comes from the SBP condition: column 0 of Q sums to -1, so
   moving the -1 to the left side gives +1.

   **Columns j = 1, ..., t-2:**

       w_0 * B[0,j] + w_1 * B[1,j] + ... + w_{r-1} * B[r-1,j] + IC(j) = 0

4. This gives **t - 1 equations total**. The system is square:

   | Scheme | p | Equations (t-1) | Weights (r) | Placeholders (last row) | Total unknowns |
   |--------|---|----------------|-------------|------------------------|----------------|
   | E4u    | 2 | 4              | 3           | 1                      | 4              |
   | E6u    | 3 | 7              | 5           | 2                      | 7              |
   | E8u    | 4 | 10             | 7           | 3                      | 10             |

   General formula: equations = t - 1 = 3p - 2, unknowns = r + n_free_per_row
   = (2p - 1) + (p - 1) = 3p - 2. Square by construction (uses r = 2p - 1).

   **Column t-1 is redundant** (linearly dependent on columns 0..t-2). Use it
   as a verification check in tests: after solving, substitute the solution
   into the column t-1 equation and assert it equals zero.

5. Unknowns: r weight symbols w_0..w_{r-1}, plus the placeholder symbols from
   the last boundary row. Together these form a linear system (the equations
   are linear in these unknowns; the alpha parameters are treated as known
   constants/parameters, not unknowns).

**Algorithm for `solve_conservation`:**

1. Collect all unknowns: w_symbols + last_row_free_symbols.

2. The equations are linear in these unknowns (the coefficients may depend on
   the global alpha symbols, but alphas are treated as parameters, not
   unknowns).

3. Extract the coefficient matrix A and constant vector b by differentiating
   each equation with respect to each unknown (or by using `linear_eq_to_matrix`).

4. Solve A * x = b using `linsolve()`. Build `solution_dict` mapping each
   unknown symbol to its solved expression.

5. Build `updated_rows`: copy `boundary_rows[0:r-1]` unchanged. For the
   last row (`boundary_rows[r-1]`), substitute the solution into its
   coefficients using `xreplace(solution_dict)` (not `subs()`).

6. Call `cancel()` on each substituted coefficient in the updated last row.

7. Return `(solution_dict, updated_rows)` where `updated_rows` has length r.

**Worked example -- E4u_1 conservation:**

Parameters: r=3, t=5, p=2. Interior stencil (h=1, indexed gamma[-2]..gamma[2]):
[1/12, -2/3, 0, 2/3, -1/12].

Boundary rows 0 and 1 are from 20.3b (symbolic in alpha_0 and alpha_1). Row 2
has 1 Taylor-system free parameter phi_0 (placeholder): its 5 coefficients are
linear in phi_0, with phi_0 appearing directly as the last coefficient
(B[2,4] = phi_0).

Interior contributions IC(j) for each column:

| Column j | Interior rows (m values) | IC(j) formula | IC(j) value |
|----------|-------------------------|---------------|-------------|
| 0        | none (j < p = 2, no interior row reaches col 0) | 0 | 0 |
| 1        | m=0 only | gamma[-2] | 1/12 |
| 2        | m=0, 1 | gamma[-1] + gamma[-2] | -2/3 + 1/12 = -7/12 |
| 3        | m=0, 1, 2 | gamma[0] + gamma[-1] + gamma[-2] | 0 - 2/3 + 1/12 = -7/12 |

The 4 conservation equations (columns 0..3, each Expr = 0):

Column 0 (IC=0, RHS from SBP = -1, so Expr includes +1):

    w_0*(6*a0 - 11)/6  +  w_1*(3*a1 - 1)/3  +  w_2*B2_0(phi_0)  +  1  =  0

Column 1 (IC = 1/12):

    w_0*(3 - 4*a0)  +  w_1*(-(8*a1 + 1)/2)  +  w_2*B2_1(phi_0)  +  1/12  =  0

Column 2 (IC = -7/12):

    w_0*(12*a0 - 3)/2  +  w_1*(6*a1 + 1)  +  w_2*B2_2(phi_0)  -  7/12  =  0

Column 3 (IC = -7/12):

    w_0*(-(12*a0 - 1)/3)  +  w_1*(-(24*a1 + 1)/6)  +  w_2*B2_3(phi_0)  -  7/12  =  0

where B2_j(phi_0) denotes row 2's j-th coefficient (a linear expression in
phi_0 from the Taylor system). This is a 4×4 linear system in (w_0, w_1, w_2,
phi_0), with coefficients depending parametrically on alpha_0 and alpha_1.

Solving with `linear_eq_to_matrix` + `linsolve` gives w_0, w_1, w_2, and
phi_0 as rational functions of alpha_0 and alpha_1. Substituting phi_0 back
into row 2's Taylor-parameterized coefficients gives the final expressions
matching E4u_1.cpp lines 90-94.

**Verification column** (col 4, IC = gamma[1] + gamma[0] + gamma[-1] +
gamma[-2] = 2/3 + 0 - 2/3 + 1/12 = 1/12):

    w_0*a0  +  w_1*a1  +  w_2*phi_0  +  1/12  =  0

This must hold automatically after substituting the solution — assert it in
the test as a consistency check (see 20.3e test case 7).

The key numerical validation: substituting the known alpha values from the
test file (alpha_0 = -0.7733323791884821, alpha_1 = 0.1623961700641681) into
the symbolic expressions must reproduce all 15 floating-point coefficients
from E4u_1.t.cpp.

**Test cases** (in `test_boundary.py`, named `test_conservation_*`):

These tests use the `e4u_pipeline` fixture from the Test File Organization
section. They verify conservation-specific structural properties; coefficient-
level validation is in 20.3e/f/g.

1. **test_conservation_weight_count_E4u:** Verify `len(w_syms) == 3` (one
   weight per boundary row) and all 3 weight symbols are keys in
   `solution_dict`.

2. **test_conservation_placeholders_resolved_E4u:** Verify the last row
   (`updated_rows[2]`) has no `phi_*` symbols remaining. For each coefficient
   in `updated_rows[2].coefficients`, check that
   `expr.free_symbols <= {alpha_0, alpha_1}`.

3. **test_conservation_redundant_column_E4u:** Compute the weighted column
   sum for column t-1 = 4 (the redundant column not used in the solve).
   Verify it equals zero after `cancel()`, confirming the system is
   over-determined-consistent:
   ```python
   w_exprs = [solution_dict[w] for w in w_syms]
   col_sum = sum(w * row.coefficients[4]
                 for w, row in zip(w_exprs, updated_rows))
   col_sum += _interior_contribution(4, result.r, 2, result.interior_coeffs)
   assert cancel(col_sum) == 0
   ```
