## Background

The uniform boundary stencil (Module 2) produces an r x t coefficient matrix where each entry is a rational function of free alpha parameters. When an embedded boundary intersects the grid at fractional position psi in (0,1], the stencil must account for the wall point that sits at non-uniform spacing from the regular grid points.

The TEMO construction (Brady & Livescu, 2021) grows the stencil from r x t to (r+1) x (t+1) — one extra row and one extra column for the wall/degenerate point — and makes all coefficients rational functions of both psi and alpha. At psi=1, the cut-cell stencil recovers the uniform stencil. As psi -> 0, the wall point coincides with the first grid point and the stencil degenerates gracefully.

Key observation from the existing C++ code: `E2_1.cpp` has `R=4, T=5` (the cut-cell dimensions) while the uniform E2 first-derivative boundary would have `r=3, t=4` — confirming the +1 growth in both dimensions. The `nbs_floating` method takes `psi` as an argument and produces coefficients as rational functions of psi and the 4 free `alpha[0..3]` parameters.

### 20.5a — Scheme parameters and dimensions (Steps 1-2)

Choose the scheme parameters from Table 1 of Brady & Livescu (2021) and compute the stencil dimensions using Eq. 11.

**Step 1 -- Scheme parameters:**

Each scheme is specified by four values: `(p, q, s, nextra)`.

| Parameter | Meaning |
|-----------|---------|
| p | Interior half-width (RHS bandwidth) |
| q | Boundary accuracy order |
| s | LHS half-width (0 for explicit, 1 for tridiagonal compact) |
| nextra | Extra rows/columns for numerical optimization |

From Table 1 of Brady & Livescu (2021):

| Scheme | nu | p | q | s | nextra |
|--------|-----|---|---|---|--------|
| E2_1 | 1 | 1 | 1 | 0 | 1 |
| E2_2 | 2 | 1 | 1 | 0 | 0 |
| E4_1 | 1 | 2 | 3 | 0 | 0 |
| E4_2 | 2 | 2 | 3 | 0 | 0 |

**Step 2 -- Dimension formulas (Eq. 11a, 11b):**

The _uniform_ boundary stencil has r rows and t columns:

```
t = p + q + 1 + nextra          (Eq. 11a -- stencil width)
r = q + 1 + nextra              (Eq. 11b -- number of boundary rows)
```

The _cut-cell_ stencil adds one row (the last boundary row that can use a near-centered stencil) and one column (the wall point):

```
R = r + 1       (cut-cell rows)
T = t + 1       (cut-cell columns, including wall)
```

**Worked examples:**

```
E2_1:  p=1, q=1, nextra=1
       t = 1 + 1 + 1 + 1 = 4,   r = 1 + 1 + 1 = 3     (uniform: 3x4)
       T = 5,                    R = 4                   (cut-cell: 4x5)
       X = 0                                             (no Neumann rows)
       => matches E2_1.cpp: P=1, R=4, T=5, X=0

E2_2:  p=1, q=1, nextra=0
       t = 1 + 1 + 1 + 0 = 3,   r = 1 + 1 + 0 = 2     (uniform: 2x3)
       But E2_2.cpp has R=2, T=4. The uniform stencil is
       actually 1x3 (r=1, t=3). The discrepancy: for 2nd
       derivatives, the last uniform boundary row coincides
       with the first interior row, so the _effective_ uniform
       boundary has r_eff = r - 1 = 1 rows.
       Cut-cell: R = r_eff + 1 = 2, T = t + 1 = 4.
       X = 2                                             (Neumann coupling rows)
       => matches E2_2.cpp: P=1, R=2, T=4, X=2

E4_1:  p=2, q=3, nextra=0
       t = 2 + 3 + 1 + 0 = 6,   r = 3 + 1 + 0 = 4     (uniform: 4x6)
       T = 7,                    R = 5                   (cut-cell: 5x7)

E4_2:  p=2, q=3, nextra=0
       t = 2 + 3 + 1 + 0 = 6,   r = 3 + 1 + 0 = 4     (uniform: 4x6)
       r_eff = 3, R = 3 + 1 - 1 = 3, T = 6 + 1 = 5... but actually
       E4_2.cpp has R=3, T=5, X=3. So r_eff = R - 1 = 2.
       => matches E4_2.cpp: P=2, R=3, T=5, X=3
```

Note: The Neumann extra row count X equals the number of boundary rows that need derivative-coupling terms: `X = R` for 2nd-derivative stencils, `X = 0` for 1st-derivative stencils.

### 20.5b — Uniform boundary stencil B^u_l (Step 3)

Derive the r x t uniform boundary stencil from Module 2. This item does not add new code -- it documents how the existing Module 2 output feeds into the TEMO pipeline and specifies the exact interface contract.

**Module 2 output (from 20.3):**

The uniform boundary derivation produces an r x t coefficient matrix `B^u_l` where each entry `alpha^u_{i,j}` is a rational function of free alpha parameters. The system has `q+1` Taylor accuracy constraints per row and `t` unknowns, leaving `t - (q+1)` free parameters per row before conservation. Conservation (Section 3.3 of 2019 paper) fixes some of these, leaving the final free alpha count.

**Interface contract with TEMO:**

The TEMO module (20.5c-20.5d) receives from Module 2:
1. The coefficient matrix `B^u_l` (r x t, entries are rational in alpha symbols)
2. The interior stencil coefficients `gamma_{-p}, ..., gamma_p` (length 2p+1)
3. The quadrature weights `w_0, ..., w_{r-1}` (rational in alpha)
4. The list of remaining free alpha symbols

**Worked example -- E2_1 uniform boundary (r=3, t=4):**

```
B^u_l = | alpha^u_{0,0}  alpha^u_{0,1}  alpha^u_{0,2}  alpha^u_{0,3} |
        | alpha^u_{1,0}  alpha^u_{1,1}  alpha^u_{1,2}  alpha^u_{1,3} |
        | alpha^u_{2,0}  alpha^u_{2,1}  alpha^u_{2,2}  alpha^u_{2,3} |
```

Each row satisfies the Taylor system (q+1=2 equations, t=4 unknowns):

```
Row i:  V_uniform[i] @ alpha^u_i = e_nu
        where V_uniform[i] is the 2x4 Vandermonde for points [0, 1, ..., 3] about x_i
```

Interior stencil: `gamma = [-1/2, 0, 1/2]` (central difference, p=1).

**Worked example -- E2_2 uniform boundary (r=1, t=3):**

```
B^u_l = | alpha^u_{0,0}  alpha^u_{0,1}  alpha^u_{0,2} |
```

Single row with q+1=2 equations and t=3 unknowns, leaving 1 free parameter before conservation. Interior stencil: `gamma = [1, -2, 1]` (central difference, p=1, nu=2).

### 20.5c — Degenerate stencil B^d_l via Design Principles 1 and 2 (Step 4)

Construct the degenerate (psi=0) stencil `B^d_l`, an `(r+1) x (t+1)` matrix operating on `[f_wall, f_0, f_1, ..., f_{t-1}]`. The degenerate stencil is fully determined by `B^u_l` and the variant choice.

**Design Principle 1 -- Truncation Error Matching (TEM):**

At psi=0 the wall point coincides with x_0, so the stencil must degenerate gracefully. The TEM constraint requires that the leading truncation error of the cut-cell stencil match that of the uniform stencil:

```
alpha^d_{i,wall} + alpha^d_{i,0} = alpha^u_{i,0}     for i = 0, ..., r-1
alpha^d_{i,j}    = alpha^u_{i,j}                      for j >= 1, i = 0, ..., r-1
```

This means columns j >= 1 of `B^d_l` (rows 0..r-1) are exactly the corresponding columns of `B^u_l`. The wall column and column-0 together absorb the uniform column-0 value, with one degree of freedom per row (the split between `alpha^d_{i,wall}` and `alpha^d_{i,0}`).

**Design Principle 2 -- Variant selection:**

The variant resolves the wall/column-0 split. Three choices:

| Variant | Name | Definition | Effect |
|---------|------|------------|--------|
| B^{d,0} | 0th-order continuity | `alpha^d_{i,wall} = alpha^u_{i,0}`, `alpha^d_{i,0} = 0` | Wall gets full weight; point-0 is zeroed |
| B^{d,1/2} | 1st/2nd-order continuity | `alpha^d_{i,wall} = 0`, `alpha^d_{i,0} = alpha^u_{i,0}` | Wall is zeroed; point-0 gets full weight |

Variant mapping to stencil type:

```
1st derivative (nu=1):  always B^{d,0}   (variant=0)
2nd derivative (nu=2):  B^{d,1/2}        (variant=1)
```

For 1st derivatives, `B^{d,0}` is used because the wall value participates directly in the upwind stencil. For 2nd derivatives, `B^{d,1/2}` is used because the wall value is handled separately (Dirichlet: known; Neumann: coupled via eta).

**Last row (row r) -- the near-interior row:**

Row r of `B^d_l` is the first interior stencil shifted to the boundary. Its coefficients come from the interior scheme `gamma_{-p}, ..., gamma_p` placed at the appropriate position. At psi=0 the wall column entry is:

```
alpha^d_{r,wall} = gamma_{-p}     for B^{d,0}
alpha^d_{r,wall} = 0              for B^{d,1/2}
```

The remaining entries in row r are the interior coefficients shifted by one position (to account for the wall column).

**Worked example -- E2_1 B^{d,0} (4x5):**

Let `a_{ij}` denote `alpha^u_{i,j}`. Interior: `gamma = [-1/2, 0, 1/2]`.

```
B^{d,0} = | a_{0,0}    0        a_{0,1}    a_{0,2}    a_{0,3}  |   row 0
           | a_{1,0}    0        a_{1,1}    a_{1,2}    a_{1,3}  |   row 1
           | a_{2,0}    0        a_{2,1}    a_{2,2}    a_{2,3}  |   row 2
           | -1/2       0          0        1/2          0       |   row 3 (interior)
              wall      col-0     col-1     col-2       col-3
```

Row 3 places the interior stencil `[-1/2, 0, 1/2]` centered at i=3 over points [wall, 0, 1, 2, 3]. Since the stencil spans i-p..i+p = 2..4, only columns for x_2 and x_4 would be nonzero on a uniform grid. On the cut-cell grid, `gamma_{-p} = gamma_{-1} = -1/2` maps to the column for x_2 (col index 3 in the 5-column layout), and `gamma_0 = 0` maps to x_3 (col 4), `gamma_1 = 1/2` maps to x_4 (outside the stencil width, so it vanishes). The wall column gets `gamma_{-p}` scaled by psi in the general case but equals `gamma_{-p}` at psi=0 for B^{d,0}.

**Worked example -- E2_2 B^{d,1/2} (2x4):**

Let `a_{0,j}` denote `alpha^u_{0,j}`. Interior: `gamma = [1, -2, 1]`.

```
B^{d,1/2} = | 0        a_{0,0}    a_{0,1}    a_{0,2}  |   row 0
             | 0        gamma_{-1} gamma_0    gamma_1   |   row 1 (interior)
               wall     col-0      col-1      col-2

            = | 0        a_{0,0}    a_{0,1}    a_{0,2}  |
              | 0        1          -2         1         |
```

Wall column is zeroed (B^{d,1/2} variant). Row 1 places interior `[1, -2, 1]` at positions [0, 1, 2].

  - Rows 0..r-1: apply Design Principles 1 and 2 to split `B_u` column-0

### 20.5d — B_l(psi) general cut-cell stencil construction (Step 5)

Construct the psi-parameterized cut-cell stencil `B_l(psi)`, an `(r+1) x (t+1)` matrix of rational functions of psi (and alpha). This is the central step of the TEMO procedure.

**Grid layout and non-uniform spacing:**

```
    wall          x_0       x_1       x_2       ...       x_{t-1}
     |--- psi*h ---|--- h ---|--- h ---|--- h ---|
```

Normalized deltas from row center x_i:

```
delta_{i,wall} = -(psi + i)         (psi-dependent, column 0)
delta_{i,j}    = (j - i)            (integer, columns 1..t, for grid point x_j)
```

Only column 0 introduces psi into the Vandermonde matrix.

**Taylor accuracy system per row (non-uniform):**

For each row i, the Vandermonde-like system has `q+1` equations and `t+1 = T` unknowns:

```
V[k, 0]   = (-(psi + i))^k / k!    for k = 0, ..., q       (wall column)
V[k, j+1] = (j - i)^k / k!         for k = 0, ..., q       (regular columns j=0..t-1)
rhs[k]    = 1 if k == nu, else 0
```

This system is underdetermined (T unknowns, q+1 equations). The free parameters are pinned by the psi-interpolation rules below.

**Psi-interpolation rules (Section 3.2):**

The cut-cell stencil `B_l(psi)` must satisfy two limits:
- `B_l(1)` recovers the uniform stencil `B^u_l` (embedded in the (r+1)x(t+1) frame)
- `B_l(0)` recovers the degenerate stencil `B^d_l` (from 20.5c)

The coefficients are split into three categories:

**Category A -- Column being zeroed (delta column, col 0 = wall):**

For rows i = 0, ..., r-1:

```
alpha_{i,0}(psi) = psi * alpha^u_{i,0}
```

This linearly sends the wall coefficient to zero as psi -> 0 (for B^{d,0}) or to alpha^u_{i,0} as psi -> 1. It is the "column being zeroed" in the paper's terminology because at psi=0 the wall point merges with x_0.

For row r (last row):

```
alpha_{r,0}(psi) = psi * gamma_{-p}
```

where `gamma_{-p}` is the leftmost interior coefficient. (At psi=0, the wall column vanishes; at psi=1, it equals the interior coefficient.)

**Category B -- Extra free parameters (from nextra > 0):**

When nextra > 0, there are extra entries whose values are not forced by Taylor accuracy alone. These interpolate between the uniform and degenerate limits:

```
alpha_{i,j}(psi) = psi * alpha^u_{i,j} + (1 - psi) * alpha^u_{i-1,j-1}
```

This applies when both `(i-1)` and `(j-1)` are valid indices into B^u_l (i.e., `0 <= i-1 <= r-1` and `0 <= j-1 <= t-1`). When i=0 or j corresponds to column 0, the `(1-psi)` term drops out and the rule reduces to Category A.

The key insight: these extra parameters exist because nextra added rows/columns beyond what Taylor accuracy requires. Their psi-dependence is chosen to smoothly connect the uniform and degenerate limits.

**Category C -- Constrained coefficients:**

All remaining entries (not in Category A or B) are determined by solving the Taylor accuracy system. For each row i, after substituting the known values from Categories A and B into the q+1 Taylor equations, the remaining unknowns are fully determined. The solve produces rational functions of psi.

**Solve procedure for each row i = 0, ..., r:**

1. Build the non-uniform Vandermonde system: `V` is `(q+1) x (t+1)`, `rhs` is `(q+1) x 1`
2. Identify Category A/B entries for this row -- these are known functions of psi
3. Move known entries to the RHS: `rhs_reduced = rhs - V[:, known_cols] @ known_values`
4. Solve the reduced system `V[:, free_cols] @ x = rhs_reduced` for the free entries
5. Each free entry is a rational function of psi (in QQ(psi) -- see 20.5e)

**Worked example -- E2_1, row 0 (i=0):**

Vandermonde (2 equations, 5 unknowns):

```
V = | 1       1    1    1    1  |     rhs = | 0 |
    | -psi    0    1    2    3  |           | 1 |
```

Category A: `alpha_{0,0}(psi) = psi * alpha^u_{0,0}` (col 0 = wall)
Category B: none for i=0 (no valid i-1 index)
Free: cols 1..4 (4 unknowns), but only 2 equations remain after substituting col 0

After moving col 0 to RHS:

```
| 1    1    1    1 | @ [c1, c2, c3, c4] = | -psi * alpha^u_{0,0}         |
| 0    1    2    3 |                       | 1 + psi^2 * alpha^u_{0,0}    |
```

This is underdetermined (4 unknowns, 2 equations) -- 2 entries remain free. These become the runtime alpha parameters after conservation (20.5f).

**Worked example -- E2_2, row 0 (i=0):**

Vandermonde (2 equations, 4 unknowns, nu=2):

```
V = | 1       1      1      1     |     rhs = | 0 |
    | -psi    0      1      2     |           | 0 |
    | psi^2/2 0      1/2    2     |           | 1 |
```

Wait -- for E2_2, q=1 means only q+1=2 Taylor constraints... but nu=2 requires matching through order 2. The boundary order q must satisfy `q >= nu` for the derivative to appear in the Taylor system. For E2_2: `q = 2*1 - 1 = 1`... but we need to match through at least order nu=2.

Actually, re-examining: the Taylor system matches through order q, and the RHS has a 1 at position nu. For this to be valid, q >= nu. For E2_2 with nu=2, we need q >= 2. The paper uses q = 2p + 2s - 1 for the boundary order. With p=1, s=0: q = 1. But that gives only 2 rows in V, and position nu=2 would be out of bounds!

The resolution: for 2nd derivatives on explicit schemes, the Taylor system has `t` columns and the order must be at least nu. The system is built with enough rows to include the derivative being approximated. The effective number of matching conditions is `max(q+1, nu+1)`. For E2_2: `max(2, 3) = 3` rows.

With 3 equations and 4 unknowns (T=4), there is 1 free parameter per row.

```
V = | 1       1      1      1     |     rhs = | 0 |
    | -psi    0      1      2     |           | 0 |
    | psi^2/2 0      1/2    2     |           | 1 |
```

Category A: `alpha_{0,0}(psi) = psi * alpha^u_{0,0}` (col 0 = wall, using B^{d,1/2}: wall=0 at psi=0 means `alpha_{0,0}(psi) = psi * 0 = 0`... but wait, for B^{d,1/2} the wall column is zero at psi=0, while the uniform limit at psi=1 has `alpha^u_{0,wall}` = 0 (no wall column in uniform). So `alpha_{0,0}(psi) = psi * (some value)`.

The precise handling depends on the variant. For B^{d,1/2}:
- At psi=0: wall=0
- At psi=1: the wall column disappears into the uniform stencil, so wall=0

For B^{d,0}:
- At psi=0: wall = alpha^u_{i,0}
- At psi=1: wall = alpha^u_{i,0} (continuous)

So for B^{d,0}: `alpha_{i,wall}(psi) = alpha^u_{i,0}` (constant, not psi-dependent!)

Correction: the formula `alpha_{i,wall}(psi) = psi * alpha^u_{i,0}` applies specifically in the paper's formulation where the wall column is the "delta" column being zeroed. The exact interpolation depends on which column is being zeroed and the variant.

### 20.5e — Cut-cell conservation and QQ(psi) field (Steps 5-6)

Apply the extended SBP conservation constraints (Section 3.3 of Brady & Livescu 2021) to the cut-cell stencil, and implement the QQ(psi) fraction field arithmetic that makes this tractable.

**Extended conservation constraint (Section 3.3):**

The SBP property for the cut-cell stencil requires:

```
sum_{i=0}^{R-1} w_i * alpha_{i,j}(psi) = 0    for interior columns j = p+1, ..., T-1
```

where the psi-dependent quadrature weights are:

```
w_0(psi) = psi         (weight for the cut cell -- proportional to its volume fraction)
w_i      = 1           for i = 1, ..., R-1   (full cells have unit weight)
```

There is one additional constraint that ties column 0 (the wall/cut-cell column) to the uniform conservation:

```
w_0 * alpha_{0,0}(psi) = w^T_u * B^u_{l,0}
```

where `w^T_u * B^u_{l,0}` is the weighted sum of the uniform stencil's first column. This ensures the cut-cell norm smoothly extends the uniform norm.

Together, these constraints form a linear system in QQ(psi) (since `w_0 = psi` introduces psi-dependence). Each constraint fixes one free parameter.

**For stencils without free alpha parameters (E2_2):**

After TEMO construction + conservation, all coefficients are fully determined as rational functions of psi. Example from `E2_2.cpp`:

```
c[4] = (2 + 4*psi) / (2 + 3*psi + psi^2)
c[6] = (-2 + 4*psi^2) / (1 + psi)
c[7] = (2 - 2*psi^2) / (2 + psi)
```

These are the exact rational functions that the derivation must produce.

**For stencils with free alpha parameters (E2_1):**

Conservation reduces the free parameter count. E2_1 starts with more free parameters from the underdetermined Taylor systems; after conservation, exactly 4 remain (`alpha[0..3]`), matching `std::array<real, 4> alpha` in `E2_1.cpp`.

**QQ(psi) fraction field -- performance-critical implementation:**

All arithmetic in 20.5d and 20.5e must use the SymPy fraction field `QQ(psi)` to avoid catastrophic slowdown from `cancel()` calls.

The problem: after solving the TEMO system, each coefficient is a rational expression in psi. Naive SymPy simplification (`cancel(expr)`) must factor multivariate polynomials and is catastrophically slow:

```python
# NAIVE (24 seconds per expression at E4 scale):
result = cancel(expr.xreplace(solution))
```

The solution: work in `QQ(psi)`, where arithmetic automatically maintains reduced form:

```python
from sympy import QQ, Symbol, cancel

psi = Symbol('psi')
K = QQ.frac_field(psi)

# Convert SymPy -> field element (cancel once on input)
def to_field(expr):
    return K.from_sympy(cancel(expr))

# All intermediate arithmetic is exact and reduced -- no cancel needed
a = to_field(some_expr)
b = to_field(other_expr)
c = a + b * K.from_sympy(psi)

# Convert back to SymPy at the end
result_sympy = K.to_sympy(c)
```

| Operation | Naive SymPy | QQ(psi) field |
|-----------|-------------|---------------|
| Single substitution + cancel | 24s | 0.001s |
| Full E2_1 stencil (4x5 = 20 entries) | ~8 min | <0.05s |
| Full E4_2 stencil (3x5 = 15 entries) | ~6 min | <0.1s |

**When alpha parameters are present (E2_1):**

The field is `QQ(psi)`, not `QQ(psi, alpha_0, ...)`. Alpha symbols remain as SymPy `Symbol` objects. The coefficient for each matrix entry has the form:

```
alpha_{i,j}(psi) = R_0(psi) + R_1(psi)*alpha_0 + R_2(psi)*alpha_1 + ...
```

where each `R_k(psi)` is a rational function in QQ(psi). The solve decomposes the solution into a psi-rational part (processed via QQ(psi)) and alpha-polynomial coefficients (kept as SymPy). At codegen time, the alpha values are C++ runtime variables.

**Implementation:**

- [ ] `field_solve(V, rhs, prescribed, psi)` -> solution vector with entries in QQ(psi)
  - Convert V entries to QQ(psi) field elements
  - Perform Gaussian elimination in the field
  - Return solution as field elements (or SymPy exprs for alpha-dependent terms)
- [ ] `build_cut_cell_conservation(B_l_psi, p, psi)` -> constrained stencil + weights

  - Solve in QQ(psi) for constrained alpha values
  - Return: fully constrained coefficient matrix and norm weights `[psi, 1, 1, ..., 1]`
- [ ] Verify: E2_2 conservation produces the exact rational expressions from E2_2.cpp

### 20.5f — Neumann eta coefficients and output assembly (Step 7)

For 2nd-derivative stencils with Neumann BC support (E2_2 with X=2, E4_2 with X=3), derive the eta coupling coefficients that relate the prescribed wall derivative to the stencil rows.

**Neumann BC formulation (Eq. 8 of Brady & Livescu 2021):**

When the wall derivative `f'(x_wall)` is prescribed (Neumann BC), it provides an additional known quantity. The stencil for row i is modified to:

```
f''(x_i) ~= (1/h^2) * [ sum_{j=0}^{T-1} alpha^N_{i,j} * f(x_j) + eta_i * h * f'(x_wall) ]
```

where `eta_i` are the Neumann coupling coefficients. The `eta_i * h * f'(x_wall)` term uses the known wall derivative to improve accuracy. The factor of h (not h^2) is because the derivative is first-order while the stencil approximates a second derivative.

There are X = R eta coefficients (one per boundary row), stored in the `x` span in the C++ output.

**Derivation of eta_i:**

The Neumann stencil is derived from the Floating stencil by replacing the wall column. In the Floating stencil, column 0 multiplies `f(x_wall)`. For Neumann BC, `f(x_wall)` is unknown but `f'(x_wall)` is known. The wall column is eliminated by:

1. Express `f(x_wall)` via Taylor expansion about x_0: `f(x_wall) = f(x_0) - psi*h*f'(x_0) + ...`
2. Replace `f'(x_0)` using the Neumann BC value `f'(x_wall)` plus corrections
3. The resulting coupling coefficient `eta_i` absorbs the wall derivative contribution

Alternatively, the Neumann stencil can be derived directly by including `h * f'(x_wall)` as an extra "virtual column" in the Taylor system and solving for both the regular coefficients and the eta coefficients simultaneously.

**C++ output structure (from E2_2.cpp):**

```cpp
std::span<const real>
nbs_neumann(real h, real psi, std::span<real> c, std::span<real> x, bool right) const
{
    // x[0..X-1] = eta coefficients (divided by h, not h^2)
    x[0] = -2;
    x[1] = 2 * t18;   // t18 = psi - 1

    // c[0..R*T-1] = modified stencil coefficients
    c[0] = (4 + 8 * psi) * -1 * 1.0 / (2 + 3 * psi + t9);
    ...

    for (auto&& v : c) v /= (h * h);
    for (auto&& v : x) v /= h;         // NOTE: x divided by h, not h^2

    if (right) {
        std::ranges::reverse(c);
        std::ranges::reverse(x);
        for (auto&& v : x) v *= -1;    // eta sign flip for right boundary
    }
}
```

Key observations:
- The `x` array has X entries, divided by `h` (1st-order scaling)
- The `c` array has `R * T` entries, divided by `h^2` (2nd-order scaling)
- Right boundary: reverse both c and x; negate only x (not c, since 2nd derivative is symmetric)

**Worked example -- E2_2 Neumann at psi=0:**

From E2_2.t.cpp: `x = [-4, -4]` at h=0.5, psi=0 (before h-scaling: `x_raw = [-2, -2*(0-1)] = [-2, 2]`, then `x /= h = x / 0.5` gives `[-4, 4]`... but test shows `[-4, -4]`). The exact values depend on the full derivation.

**Output assembly (CutCellResult):**

After computing the floating stencil (20.5d-e) and Neumann eta coefficients (this item), assemble the three output variants:

```python
class CutCellResult:
    floating:  Matrix      # R x T, entries are f(psi, alpha)
    dirichlet: Matrix      # (R-1) x T, rows 1..R-1 of floating (wall row dropped)
    neumann:   Matrix      # R x T main + X eta entries (only for nu=2)

```

The Dirichlet stencil drops the wall row (row 0) because the wall value is known. The wall column contributions are absorbed into the RHS at runtime (the C++ code handles this). In the C++ output, `nbs_dirichlet` writes `(R-1) * T` values.

**Implementation:**

- [ ] `derive_neumann_eta(floating_stencil, p, q, nu, psi)` -> list of X eta expressions
  - For each boundary row i = 0, ..., R-1: derive eta_i from the wall column and Taylor expansion

- [ ] `assemble_cut_cell_result(floating, neumann_main, eta, R, T, X)` -> `CutCellResult`

## Key Implementation Notes

1. **psi is always a SymPy Symbol**, never a float, during derivation. Numerical evaluation happens only in tests and codegen.

2. **The h normalization is deferred to codegen.** All derivation works with h=1 (normalized deltas). The `1/h` or `1/h^2` factor is applied in the generated C++ (matching the existing pattern: `for (auto&& v : c) v /= h;`).

3. **Right-boundary generation is not a separate derivation.** The left boundary stencil is derived once; the right boundary is obtained by the reversal rule in codegen:
   - 1st derivative: negate all coefficients, reverse order
   - 2nd derivative: reverse order only (no sign change)
   This matches the existing C++ pattern (`if (right) { for (auto&& v : c) v *= -1; std::ranges::reverse(c); }`).

4. **Dirichlet stencil from Floating stencil.** The Dirichlet variant has one fewer row (the wall row is eliminated because the wall value is known). In the C++ output: `nbs_dirichlet` has `(R-1) * T` entries while `nbs_floating` has `R * T` entries.

5. **Neumann support (2nd derivative only).** Stencils like E2_2 and E4_2 have `X > 0` extra rows for Neumann BC. The Neumann stencil has the same `R * T` main coefficients plus `X` extra entries that couple to the derivative value at the wall. The extra entries go into the `x` span parameter of `nbs()`. This is visible in `E2_2.cpp` lines 153-181 where `nbs_neumann` writes both `c[0..7]` and `x[0..1]`.

6. **E2_1 alpha parameters.** The struct has a single `std::array<real, 4> alpha` array with 4 free parameters (`alpha[0..3]`). The derivation pipeline produces coefficient expressions parameterized by these; codegen maps them to `alpha[i]` array accesses.

7. **polyE2_1 is out of scope.** It has interpolation constraints (separate `fa[6]`, `da[3]`, `ia[4]` parameter arrays) that go beyond the TEMO procedure described in Brady & Livescu 2021. Supporting polyE2_1 would require extending the pipeline with additional interpolation constraint machinery — a future extension after the core TEMO pipeline is validated against E2_1 and E2_2.
